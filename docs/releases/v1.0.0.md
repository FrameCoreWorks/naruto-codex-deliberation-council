# Codex Deliberation Council 1.0.0

Release date: 2026-07-18

Version `1.0.0` is the first stable package-contract release of the request-only
Naruto shadow-clone deliberation protocol for Codex.

## Release Contract

The release freezes:

- exact, case-sensitive first-token activation with `$naruto`
- four Naruto runtime IDs sharing `actor_identity_id: naruto_uzumaki`
- four fixed methods: Integrator, Challenger, Strategist, and Empirical
  Verifier
- Kakashi guidance and reveal moderation
- Yamato identity, packet, boundary, and phase-safety control
- Tsunade Senju as the public identity of the Hokage role assumed by the parent
  Codex process
- one blind commit barrier, one common reveal, and one revision in each
  original instance thread
- read-only child profiles, fail-closed capability checks, packet schemas,
  checkpoint chain, installer roots, package layout, and checksums
- neutral optional evidence-preparation roles plus a host-provided independent
  `final_qa` contract for consequential results

Tsunade is not a seventh child profile or an additional solver. The four Naruto
profiles are one actor identity trained through different methods, not four
different character personas.

## Empirical Verifier

The fourth method keeps its stable runtime ID `naruto_clone_verifier` and
method ID `naruto_empirical_method.v1`. Its public name is now
`Naruto Clone: Empirical Verifier`.

It must produce a complete solution route organized around hypotheses,
discriminating tests, observable evidence, decision thresholds, degraded or
counterfactual cases, and fallback or rollback conditions. It does not change
shared acceptance criteria, judge peers, issue final status, execute tests, or
invent test outcomes.

## Installation And Validation

Node.js 22 or newer is required for the local helpers. The package has no
third-party runtime dependencies.

```bash
npm test
npm run checksums:check
```

Preview installation into an existing project before writing:

```bash
node scripts/install.mjs --scope project --target /absolute/path/to/project --dry-run
node scripts/install.mjs --scope project --target /absolute/path/to/project
```

The installer updates `0.4.x` package-owned paths in place only when reviewed.
It does not delete files, create a Git remote, push, upload, publish, or create
a release tag.

## Acceptance Boundary

The stable claim applies to the repository files, schemas, fixtures, installer,
checksums, and isolated installation behavior. It does not assert that every
Codex host build can discover all six custom profiles or re-contact the same
four instance threads after reveal.

Host custom-profile and same-thread acceptance remains a target-environment
check. If the capability cannot be demonstrated, the protocol returns
`blocked`. See
[`../release-acceptance-v1.0.0.md`](../release-acceptance-v1.0.0.md).
