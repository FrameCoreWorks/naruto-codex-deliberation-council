# Naruto Codex Deliberation Council 1.1.0

Release date: 2026-07-19

Version `1.1.0` is a protocol-consistency and audit hardening release. It
preserves the exact `$naruto` trigger, six runtime IDs, four method IDs, one
Naruto actor identity, Tsunade parent-Hokage role, and standalone layout.

## Fixed Provenance Contract

Mandatory references no longer require model-visible opaque thread-handle
hashes. Revision continuity consistently depends on the parent-retained target
returned at spawn and a successful host-tool follow-up receipt to that exact
target. The static validator rejects the stale positive phrases that caused the
`1.0.1` contradiction.

The parent-owned logical sidecar is now described consistently as recording and
binding parent-observed state. It is not host-enforced, durable, or tamper
evident and does not by itself prove a live run.

## Blind Semantic-redundancy Audit

Kakashi now records a pre-reveal, claim-level comparison of blind candidates:

- pairwise material relation
- overlapping and unique claim references
- evidence-lineage deltas
- unique verified contributions
- whether core claims and evidence lineages are materially equivalent

Only verified differences in both claim meaning and evidence lineage plus a
candidate-specific unique verified contribution are `sufficient`. A difference
in one dimension, redundant outputs, or an unverifiable comparison caps the
result at `provisional_consensus`. The audit forbids inferring
diversity from method IDs or wording differences and forbids invented similarity
percentages. It also separates blind redundancy from evidence-backed
post-reveal convergence.

Static fixtures require a complete unordered pair matrix, claim/evidence
reference integrity, and consistent status/result-ceiling combinations. They
also reject a missing pair, an unknown reference, and a falsely sufficient
status over fully redundant outputs.

This is a bounded moderator heuristic. It is not a deterministic metric,
benchmark result, or evidence that the four methods outperform a single-agent
or planner-critic baseline. Behavioral quality lift remains `NOT MEASURED`.

## Final-QA Interoperability

The consensus template now includes non-bundled request/result message shapes
for a host-provided independent `final_qa` reviewer. The example preserves role
blindness and reproducible findings without creating a seventh runtime profile
or claiming the reviewer exists on a target host. It binds request ID, task ID,
request digest, final-artifact digest, result digest, same-task manifest, and
effective outcome; mismatch or replay is blocked and covered by negative
validation probes. Closed request/result/finding envelopes and the reviewed
artifact reject role-bearing or raw-reasoning payloads.

The artifact digest uses a frozen consensus QA-review projection that omits
only the post-QA result block and final manifest hash. Filling those two fields
does not create a circular digest dependency; any other edit invalidates QA.
The frozen `result_status` records the deliberation proposal;
`final_qa.effective_result_status` preserves it only for `pass` or
`not_required` and otherwise blocks delivery.

## Publication And Rights Status

The naming-risk record now states the factual current route: franchise-derived
names and the user-selected fan-art banner remain in the public package with
status `NOT CLEARED`. Explicit publisher legal-risk confirmation remains
pending and is not inferred from asset selection or technical publication.
Re-review triggers cover commercial distribution, paid promotion,
merchandising, material branding expansion, and rights-holder notice.

## Acceptance Boundary

The repository package self-check, installer suite, checksums, and exact-release
CI must pass. This remains a package self-check, not an arbitrary live run or a
validator for arbitrary recorded host artifacts.

The installer continues to validate the installed target with a private
verified validator copy before reporting success.

Target-host runtime remains `NOT PROVEN` until a target build records discovery,
effective permissions, capacity, fresh-context spawning, exact-target receipts,
checkpoint state, redundancy disposition, and required final QA. The package
still does not bundle a host adapter, tamper-evident event log, benchmark
results, rollback, or uninstall.
